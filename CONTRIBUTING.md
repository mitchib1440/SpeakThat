# AI-Coded Contributions
As SpeakThat started life as an AI experiment, it would be incredibly hypocritical of me to ask all contributions to be entirely human-written.
AI-written code is allowed, but with very strict quality control.

When I began creating SpeakThat in July of 2025, I was sure that nobody would use it except me. So a few bugs or broken features were fine as I was the only one to suffer from it. But SpeakThat quickly grew larger than I expected, and the consequences of including early slop code that more-or-less worked (but barely) stacked up quickly. I'm still fixing bugs left by early code and finding features that weren't properly implemented because I simply didn't bother to check it for myself, and certainly wasn't reading up on [Android's documentation](https://developer.android.com/develop#core-areas).

So while AI-assisted programming is allowed, I ask that all contributors follow these guidelines, just as I do now:
* Please ensure you have experience in development. Preferably on a project of your own or professionally for a reputable project.
* Understand every line. Do not submit blind, copy-pasted AI dumps. You must be able to explain how the code in your Pull Request works.
* Testing your own code is mandatory. Either on your own personal device or on an emulator. Preferably both.
* Prioritise privacy and security. SpeakThat handles highly sensitive notification data. Double-check that AI hasn't suggested insecure data storage, reckless intent handling, or accidental logging of private user data.
* Take full responsibility for the code you or your AI IDE contributes.
* Beware of licensing conflicts. You are responsible for ensuring any AI-generated code does not violate copyright and is fully compatible with our GPL-3.0 license.
* Leave clear code comments as much as possible. (And only in English, please!)
* Look up and reference [Android's development documentation](https://developer.android.com/develop#core-areas) where necessary.
* Keep SpeakThat's current branding. Use [Material icons](https://fonts.google.com/icons) and keep the colour schemes consistent.
